Debounce

People used to comparing Debounce with Throttle. They are both designed to avoid high-frequency triggering by instantaneous operation. Their aim is at improving the performance of web applications. But there is still a slight difference.

Concept:

The debounce function delay event until no event generated for predetermined time we set.

Also, there is another debounce function but much more rare. The event would be executed immediately once it is triggerd. However, debounce functionality would block the following events until the user stops operating to generate new events for predetermined time.

As the GIF showing, it is the first functionality. The triggerred function will display the number input below the textbox. In the debounced condition, the typed number would be shown only do we stop typing for about 1 second.

Code

const debounce = function (fn, wait) {
    let timer = null
    return function(...args) {
        if (timer) {
            clearTimeout(timer);
        }
        let that = this;
        timer = setTimeout(() => {
            fn.apply(that, args);
        }, wait);
    }
}
